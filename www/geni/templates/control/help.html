What is GENI

GENI is a web-based portal for managing Seattle resources. These resources are donated by users who have installed Seattle on their machines. In return these users are allowed to use resources donated by others. GENI enables users of donated resources to acquire nodes around the world on which they can then execute programs written in Repy, the Seattle platform programming language. GENI also lets users to share their resources with other users, and perform a variety of other tasks. For more information about about how GENI fits with the rest of the Seattle platform, see this page.

How to use the GENI portal

The GENI portal is divided into four sections, each of which can be accessed through a tab at the top of the page. Remember to maintain account security and to always Log Out of your session by clicking on the Logout link at the top right of the page.

The remainder of this section describes the various functions accessed via each of the tabs.

Profile : User keys, port, and other information

Here you you can manage your GENI account and find out meta information necessary for you to write programs for Seattle.

  • Download the toolkit. The toolkit includes the Seattle shell which you will need to run programs on your acquired vessels. For more information on how to use the Seattle shell, see this page.
  • Download the public and private keys. You will need these two key files to access your acquired vessels. Once you download them, place the two key files in the same directory as Seattle shell you downloaded with the Seattle toolkit above.

    For better security we recommend that you download and delete your private key from the server. Always store your private key in a secure location.

  • User Port. Each user is allocated a unique UDP and TCP port resources. The port number you see on this page is the port number you have to use in your Seattle programs.

My GENI : Inspecting donations, managing resource shares

This page helps you visualize and manage your Seattle resources, as well as the create, modify, and delete resource shares with other Seattle users.

  • My Credits displays the total resources you have at your disposal (number of vessels you may acquire). It also attributes these resources to either donations made by users who have installed your Seattle installers, or by users who have shared their resources with you.
  • Resource usage displays how the resources available to you are being used. Some of these will be used by you ('Me'), others will be used by users with whom you've shared your resources.
  • Managing shares. You can create new shares with Seattle users by clicking on 'share' on the bottom left of the page in the box labeled as 'Free'. You can delete existing shares, by clicking the 'X' in the share box you would like to remove. You can modify an existing share by clicking on the share box. This will allow you to change the percent of resources allocated to the share.
  • Getting more vessels. Besides sharing your resources, you make also redeem your available resources as vessels. You can do so by clicking 'get' in the box labeled as 'Free'. At this point you will see a window in which you may select the number of vessels that you would like to acquire, and the type of network environment between these vessels. There are three choices for the environment:
    • WAN : Wide area network. The vessels will be as spread out as possible.
    • LAN : Local area network. The vessels will be on the same local network.
    • Random : A random selection of vessels, and an unpredictable network environment.

My Vessels : Managing vessels -- acquiring and releasing vessel resources

On this page you can see more detailed information about your acquired vessels, such as their IP addresses, and how long you have before the vessels are released. This page also allows you to acquire more vessel resources.

  • Vessel Information. For each vessel that you acquire, you will see a record in the table on this page that tells you the vessel id which is the same id that your seash session will display when you browse your vessels. You will also see the vessel status which will tell you whether the vessel is in the correct state. Also you will see an expires in time value which indicates how long you have before the vessel will be returned back into the Seattle pool for use by other users.
  • Getting more vessels. See above for a description of how the vessel acquisition process works.

Get Donations : Donating to the platform

This page contains three links to Seattle installers for different platforms (Windows, OS X, and Linux). These installers donate the resources on a machine to your account. To simplify distribution of these installers, this page also includes a link at the top that you can email to people who might help you by installing Seattle and contributing resources on their machines on your behalf. This link will take them to a page that contains the same set of installers.
