Introduction to Beraber Donation

By running Seattle on your home computer, workstation, or computer, you are helping everyone using the Beraber framewok! Having more donations available to Beraber users implies greater flexibilities in repy applications by increasing the heterogeneity of available nodes. This enables both more accurate representations of internet (increased and varied latency times) and allows undergraduates, graduate students, and professors alike to test their hypotheses on a internetwork-like testbed without charge.

The Beraber framework ensures fair play with each donation. Our promise to you is to never use more than 10% of your computational resources (defined as RAM, CPU time, and network bandwidth; hard drive space can be set at any amount). The goals of the Beraber project are three-fold: be secure, simple, and reliable, in that order. If you decide to peruse through the code, you will see this simple philosophy echoed through many docstrings! If, by any way, you feel that this goal is being violated in the implementation of Beraber, please feel free to open a ticket on the main site (must login to do so) or drop us an e-mail on our developer's list at seattle.devel@cs.washington.edu.

Installing Beraber

Running Beraber on your machine is easy. Simply start the install script (install.py or install.sh, depending on the type of your system), and the application will do three things. (1) start the main communication program, nmmain.py (standing for node manager main thread), (2) start up the software updater, and (3) insert nmmain.py into your startup queue (crontab for linux/apple, startup items for windows).

In the unlikely event that the installation procedure is forcibly interrupted, it is necessary for you to make sure the state created by the installer is cleaned up before reattempting a reinstallation. Two things need to be checked: (1) make sure that the process is not running (check by using ps -f | grep "nmmain.py" in linux, or find pythonw.exe in the Windows task manager) and (2) that the crontab entry for Beraber is cleared (for linux, crontab -r will delete all).

Pausing Beraber

To pause the execution of Beraber, run the python script seattlestopper.py (or use the script stop_seattle.sh). To unpause, run the python script seattlestarter.py (or use the script start_seattle.sh). During the paused period, none of your computer's resources will be used.

Uninstalling Beraber

If you decide to end your donation, Beraber can be uninstalled simply by running the python script uninstall.py (or the script uninstall.sh). This process will remove Beraber from the startup queue, deactivate the main thread, and remove the execution from the user's crontab (if applicable). We would also like to know of your experiences of Beraber through donating and usage, please drop a message to our developer list! We're always striving to make a better experience for all.
